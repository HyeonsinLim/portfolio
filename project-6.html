출퇴근 지문 인식 프로그램 설계

수기로 출근 버튼을 눌러야 했던 근태 시스템을 보안 지문 데이터와 연동해 자동화하고, 피플팀의 근태 데이터 수정 건수를 0건으로 만들었습니다.

배경

출퇴근 등록이 웹 페이지 기반으로 되어 있어, 구성원이 출근 후 PC를 켜고 내부 프로그램에 로그인해 '출근하기' 버튼을 직접 눌러야만 출근으로 인정되는 구조였습니다. 이미 오피스에 도착했더라도 버튼을 늦게 누르면 지각으로 처리돼 피플팀에 데이터 수정 요청이 끊이지 않았고(하루 평균 3건, 월 평균 80건 이상), 수정이 반복될수록 근태 데이터의 신뢰도 자체가 흔들리는 상황이었습니다.

프로젝트 프로세스

01
문제
  • 수기 출근 버튼 입력 방식으로 실제 출근 시각과 시스템 기록이 어긋남
  • 근태 데이터 수정 요청이 하루 평균 3건, 월 평균 80건 이상 발생
  • 반복되는 수정으로 데이터 신뢰성 저하, 피플팀 리소스 과다 소모
02
접근
  • 전 구성원이 이미 사용 중인 보안 시스템(세콤) 지문 데이터 활용 아이디어 도출
  • 출퇴근·지각 기준, 예외 상황(엣지케이스) 대응 정책을 직접 설계
  • 개발자와 협업해 지문 데이터를 '당근서비스 워크' 근태 시스템에 자동 연계
03
실행
  • 세콤 지문 인증 데이터를 출근 시각으로 자동 반영되도록 시스템 연동
  • 출근 시각 5분 경과 후 지문 미인증 시 팀 채널에 자동 알림, 동료 간 상호 리마인드 구조 설계
04
결과
  • 피플팀의 근태 데이터 수정 건수 0건으로 감소
  • 구성원들도 출근 버튼에 신경 쓰지 않고 업무에 바로 집중할 수 있게 됨
