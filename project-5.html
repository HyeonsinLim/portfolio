당근서비스 워크 (e-HR) 개발

여러 어드민에 흩어져 있던 근태·성과·비용 정산 기능을 하나의 e-HR 시스템으로 통합하고, 개발자 의존 없이 피플팀이 직접 화면과 권한을 관리할 수 있는 구조를 만들었습니다.

배경

당근서비스는 당근마켓의 고객경험을 개선하고 대응하는 운영 자회사로, 정작 구성원들이 사용하는 인사 기능은 별도 시스템 없이 고객 문의 대응용 어드민에 함께 얹혀 있었습니다. 휴가 신청, 연장근무 신청, 법인카드 비용 정산, 성과 관리, 팀 일정 관리가 모두 한 화면 안에서 뒤섞여 있어 구성원들이 필요한 기능을 찾기 어려웠고, 인사 정보를 다루기엔 화면 구조 자체가 적합하지 않았습니다.

프로젝트 프로세스

01
문제
  • 근태·성과·비용정산 등 인사 기능이 고객대응용 어드민에 혼재
  • 전용 인사 시스템 부재로 사용성이 떨어지고 관리도 비효율적
02
접근
  • '당근서비스 워크'라는 e-HR 프로그램 신규 도입 결정
  • ERP 개발자와 협업해 구성원 관점의 화면 설계 방향 수립
  • 기능·권한 설정을 개발자가 아닌 피플팀이 직접 관리하는 구조로 전환 제안
03
실행
  • 개발자가 모든 화면을 직접 구성하려던 초기 방향을 조정, 어드민 화면에서 피플팀이 기능·권한을 직접 설정하도록 요청
  • 인사 관점의 이해가 부족한 개발자에게 필요성과 배경을 설명하며 설계 방향 설득
04
결과
  • 구성원들이 일정·성과·목표 관리를 더 쉬운 화면과 상세한 안내로 진행 가능해짐
  • 피플팀이 직접 커스터마이징하며 안내 문구 정교화, 신규 기능도 신속하게 반영 가능해짐
